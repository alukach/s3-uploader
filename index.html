    S3 Bucket Setup (for admins)
    

        
    To allow public uploads, the target S3 bucket needs three things configured:
    

        
    1. Disable Block Public Access
    
        
    
          In your bucket's Permissions tab, under
          Block public access (bucket settings), uncheck all
          options (or at minimum uncheck "Block public and cross-account access
          to buckets and objects through any public bucket or access point policy").
        
    

        
    2. Bucket Policy
    
        
    Add a bucket policy that grants public s3:PutObject access:
    
        
    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicWrite",
            "Effect": "Allow",
            "Principal": "*",
            "Action": [
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
    

        
    3. CORS Configuration
    
        
    
          Under the bucket's Permissions → Cross-origin resource
          sharing (CORS), add a configuration that allows uploads from
          your origin:
        
    
        
    [
    {
        "AllowedHeaders": ["*"],
        "AllowedMethods": ["GET", "PUT", "POST", "DELETE", "HEAD"],
        "AllowedOrigins": ["https://your-origin.example.com"],
        "ExposeHeaders": ["ETag"],
        "MaxAgeSeconds": 3600
    }
]
